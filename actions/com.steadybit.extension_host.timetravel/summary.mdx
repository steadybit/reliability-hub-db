# Introduction

Changes the clock time of a host.

> **Warning:**<br/>
> This action will may cause unwanted side effects. Use with caution.
> E.g. Other software might have hiccups when system time changes, vital access tokens might expire, etc.

# Use Cases

* Change the time of a host to a specific time.
* Test your application's ability to handle time changes, like summer winter / time changes.
* Check for certificate expiration.

# Parameters

| Parameter   | Description                                        | Default |
|-------------|----------------------------------------------------|---------|
| Duration    | How long should the clock be modified              | 30s     |
| Offset      | The offset that will be added to the current time. | 1s      |
| Disable NTP | Prevent NTP from correcting time during attack.    | true    |
