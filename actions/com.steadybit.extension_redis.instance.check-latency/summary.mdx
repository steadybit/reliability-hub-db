# Introduction

With this action, Steadybit can monitor Redis response latency and fail if a threshold is exceeded on a Redis instance.

# Use Cases

 - Detect latency spikes during chaos experiments.
 - Validate SLOs for Redis response times under failure conditions.

# Parameters

| Parameter   | Description                                | Default |
|-------------|--------------------------------------------|---------|
| Duration    | How long to monitor.                       | 60s     |
| Max Latency | Maximum allowed latency in milliseconds.   | 100     |
