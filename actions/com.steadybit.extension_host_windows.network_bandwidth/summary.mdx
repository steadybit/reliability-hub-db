# Introduction

Limit the bandwidth of outgoing network traffic.

# Prerequisites

The Extension Windows Host needs to be installed on the target system.

# Details

This attack limits the bandwidth of outgoing network connections matching the configured criteria. As a result, the combined bandwidth of multiple parallel connections might exceed the configured limit.

Without specifying a host or ip restriction, the attack would affect all traffic and may lead to broken communication between the Steadybit agent and platform. Hence, either the hostname or IP parameter is required.

# Use Cases

* Understand your service's performance under load
* Test your service's resilience to network attacks
* Test your service's resilience to decreased bandwidth

# Parameters

| Parameter         | Description                                                                       | Default  |
|-------------------|-----------------------------------------------------------------------------------|----------|
| Duration          | How long should the bandwidth be reduced?                                         | 30s      |
| Network Bandwidth | How much traffic should be allowed per second?                                    | 1024kbit |
| Hostnames         | (optional) Restrict to/from which hosts the traffic is affected.                  |          |
| IPs/CIDRs         | (optional) Restrict to/from which IP addresses or blocks the traffic is affected. |          |
| Ports             | (optional) Restrict to/from which port the traffic is affected.                   |          |
