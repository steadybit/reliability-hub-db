# Introduction

Use this attack to taint one or multiple nodes via `kubectl taint`.

# Use Cases
- Check that your pods affinity/taint/toleration configuration is working
- Simulate [Kubernetes taint-based evictions](https://kubernetes.io/docs/concepts/scheduling-eviction/taint-and-toleration/#taint-based-evictions) like a node having memory pressure

# Parameters

| Parameter | Description                                       | Default    |
|-----------|---------------------------------------------------|------------|
| Duration  | How long should the node keep tainted?            | 180s       |
| Key       | The key for the taint expression                  |            |
| Value     | The value for the taint expression                |            |
| Effect    | The effect for the taint expression               | NoSchedule |

# Rollback
The given taint is removed after the given duration or in case of an error.
