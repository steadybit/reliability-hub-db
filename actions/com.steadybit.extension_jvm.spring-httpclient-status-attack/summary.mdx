# Introduction

Return the given status code from Spring `RestTemplate` or `WebClient` calls.

# Details

This attack only targets JVMs identified as Spring Application.

The original call is not executed.

# Use Cases

* Understand how your services behave under failures in dependent services

# Parameters

| Parameter           | Description                                                                                           | Default |
|---------------------|-------------------------------------------------------------------------------------------------------|---------|
| Duration            | How long should the response be delayed?                                                              | 30s     |
| Failure Types       | (optional) What HTTP client behavior should be simulated? Will overwrite any HttpStatus configuration |         |
| Http Methods        | (optional) Which HTTP methods should be attacked?                                                     |         |
| Erroneous Call Rate | (optional) How many percent of calls should trigger an exception                                      | 100     |
| Host Address        | (optional) Which host address should be attacked                                                      | *       |
| Path Pattern        | (optional) Which URL path should be attacked? Supports Spring Ant-style path patterns (e.g. /api/**). | /**     |
