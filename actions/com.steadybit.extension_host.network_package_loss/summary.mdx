# Introduction

Cause packet loss for matching outgoing network traffic.

# Prerequisites

If you are not using our container images for the extension, you need install the `tc` (from the iproute2 package) tool to use the attack.

# Restrictions

This attack affects network traffic for protocols that use port numbers (e.g., TCP, UDP). Protocols like ICMP, which do not use ports, remain unaffected and will continue to function normally.

# Use Cases

* Understand how your services behave under specific / flaky network conditions

# Parameters

| Parameter         | Description                                                                          | Default |
|-------------------|--------------------------------------------------------------------------------------|---------|
| Duration          | How long should the traffic be lost?                                                 | 30s     |
| Hostnames         | (optional) Restrict to/from which hosts the traffic is affected.                     |         |
| IPs/CIDRs         | (optional) Restrict to/from which IP addresses or blocks the traffic is affected.    |         |
| Network Loss      | How much percentage of traffic should be lost?                                       | 70      |
| Network Interface | (optional) Target Network Interface which should be attacked. All if none specified. |         |
| Ports             | (optional) Restrict to/from which ports the traffic is affected.                     |         |
