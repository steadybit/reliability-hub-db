# Introduction

Dynamically modifies the broker's [max.connection.creation.rate](https://docs.confluent.io/platform/current/installation/configuration/broker-configs.html#max-connection-creation-rate) configuration to limit how many new client connections the broker accepts per second. The original value is automatically restored when the attack ends.

# Use Cases

 - Simulate slow connection acceptance to test how producers and consumers handle connection timeouts and retries.
 - Verify that client connection pools and retry logic behave correctly under connection throttling.

# Parameters

| Parameter                | Description                                                                                                                                                                               | Default |
|--------------------------|-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|---------|
| Duration                 | How long the configuration change stays in effect. The original `max.connection.creation.rate` value is automatically restored when the duration expires.                                 | 60s     |
| Connection creation rate | The maximum number of new connections the broker accepts per second. Set to a low value (e.g., 1) to simulate severe throttling. The original value is restored at the end of the attack. |         |
